    21  // SetLogger sets up the logging interface used throughout. The idea
    22  // here is to provide developers with the option of handling errors
    23  // and warnings in a strict way during development and a lenient way
    24  // in production. For example, in development, setting a logger that
    25  // prints a message for all logged output and panics on any logged
    26  // warning or error might be appropriate, while in production, it
    27  // would make more sense to log only warnings and errors and to
    28  // proceed without halting. All GrowthBook SDK functions leave values
    29  // in a sensible default state after errors, so production systems can
    30  // essentially ignore any errors.
    31  func SetLogger(userLogger Logger) {
    32  	logger = userLogger
    33  }
