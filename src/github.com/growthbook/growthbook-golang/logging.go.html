    35  // Logger is a common interface for logging information and warning
    36  // messages (errors are returned directly by SDK functions, but there
    37  // is some useful "out of band" data that's provided via this
    38  // interface).

    48  // SetLogger sets up the logging interface used throughout. The idea
    49  // here is to provide developers with the option of handling errors
    50  // and warnings in a strict way during development and a lenient way
    51  // in production. For example, in development, setting a logger that
    52  // prints a message for all logged output and panics on any logged
    53  // warning or error might be appropriate, while in production, it
    54  // would make more sense to log only warnings and errors and to
    55  // proceed without halting. All GrowthBook SDK functions leave values
    56  // in a sensible default state after errors, so production systems can
    57  // essentially ignore any errors.
    58  func SetLogger(userLogger Logger) {
    59  	logger = userLogger
    60  }
